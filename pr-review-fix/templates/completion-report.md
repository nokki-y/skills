# 対応完了報告テンプレート

すべての対応完了後、以下の形式で報告してください。

```markdown
## PRレビュー対応完了

### 対象PR
- PR #[番号]: [タイトル]
- ブランチ: [ブランチ名]
- URL: [PR URL]

### 対応済み
- [x] #1: path/to/file.ts L42 - 変数名を`userId`から`accountId`に修正
- [x] #2: path/to/file.ts L58 - エラーハンドリングを追加
- [x] #3: path/to/api.ts L15 - 質問に回答（コメント投稿済み）

### 未対応（理由付き）
- #4: 提案のため見送り - 現状の実装で要件を満たしているため
- #5: 別PRで対応予定 - スコープ外のリファクタリングのため

### 変更ファイル一覧
- `path/to/file.ts`
- `path/to/api.ts`

### 次のアクション
- [ ] 変更をコミット
- [ ] 再レビュー依頼
```

## 記載ルール

- 対応済みには具体的な変更内容を記載
- 未対応には必ず理由を明記
- 変更ファイル一覧でdiffの範囲を明確化
